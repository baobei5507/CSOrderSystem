# 2026-07-01 工作日志

## 修复订单管理页面三个问题

1. **实付金额显示修复**：
   - 无折扣订单统一使用 `.toFixed(2)` 格式化显示，避免整数和浮点数显示不一致
   - 提成信息（妹妹收入、客服提成）也统一 `.toFixed(2)` 格式化
   - 日期标题栏金额使用 `.toFixed(0)` 替代 `.toLocaleString()`

2. **日期编辑时区问题修复**：
   - 问题：`toISOString()` 返回 UTC 日期但 `getHours()` 返回本地时间，导致凌晨订单日期偏移一天
   - 修复：使用本地时间方法 `getFullYear()/getMonth()/getDate()` 解析日期部分
   - 后端 PUT 路由增加 number 类型预约时间处理（前端传时间戳数值）

3. **新增删除订单功能**：
   - 后端：新增 DELETE /api/orders?id=xxx 路由，含余额退还逻辑和订单快照删除
   - 前端：编辑弹窗底部添加"删除订单"按钮 + 删除确认弹窗（显示订单信息和警告）

commit: a8092fe

4. **修复取消订单错误退款问题** (commit: 55b9157):
   - 根因：`deductedBalance: body.deductedBalance ?? finalPrice` 导致非会员订单DB存了finalPrice但实际没扣余额
   - 取消时看到 `deductedBalance > 0` 就退款了，但根本没扣过
   - 修复：`deductedBalance` 只在前端明确传了且 > 0 时才存值，否则存 0

5. **修复新增顾客重复创建问题** (commit: 7d2c52b):
   - 根因：handleSubmit 没有提交防重复机制，按钮在请求期间没有禁用，双击创建两个
   - 修复：CustomersPage 和 OrdersPage 都添加 isSubmitting 状态，提交期间禁用按钮显示"提交中..."

6. **新增其他客服预约功能** (commit: 907c692):
   - DB: orders表新增 `order_source`('my'/'other') 和 `other_staff_name` 字段
   - 后端: orderSource=other 时 serviceCommission 强制为0；日报API拆分统计（myRevenue/myCommission/otherRevenue）
   - 前端创建订单: 添加「其他客服预约」开关（橙色），选中后提成自动0 + 可选填客服名
   - 订单列表: 其他客服订单显示橙色标记 + 无提成提示
   - 日报页面: 4卡片改为 总营收/我的营收/我的提成/其他客服营收
   - 目的: 股东对账追踪店铺总营收，其他客服无提成但计入总营收

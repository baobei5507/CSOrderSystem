## 系统金额单位规则（重要！）

系统中存在两种金额单位，必须严格区分：

### 元单位（直接显示，不用转换）
- `orders` 表所有金额字段：price, discount, finalPrice, totalOriginalAmount, discountAmount, deductedBalance, girlIncome, serviceCommission, storeProfit
- `packages.basePrice`
- `stores.serviceCommissionValue`
- `girls.commissionValue`（固定提成时）
- 前端输入的金额（如优惠券、充值金额输入框）

### 分单位（必须 `/ 100` 转换为元后显示）
- `customers.balance` - 余额
- `customers.totalRecharge` - 累计充值
- `recharge_records.amount` - 充值金额
- `recharge_records.giftAmount` - 赠送金额
- `balance_transactions.amount` - 流水金额

### 转换规则
- 元 → 分：`Math.round(yuan * 100)`
- 分 → 元：`fen / 100`

## 修复余额扣除单位转换问题

**问题**: 订单消费时，余额扣除金额单位错误，导致只扣了8.92元而不是892元

**原因**: `deductedBalance` 是元单位（如 892），但 `customers.balance` 是分单位（如 300000），后端扣除时直接相减导致错误

**修复**: 在 `orders.ts` 创建订单时，将 `deductedBalance` 从元转为分再扣除：
```typescript
const deductedBalanceFen = Math.round(body.deductedBalance * 100) // 元转分
```

**数据修正**: 已修正小欧的余额和余额流水记录

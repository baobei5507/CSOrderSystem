## 功能模块详情

### 1. 多店家管理（核心基础）

#### 功能说明

首页提供店家切换按钮，支持多店家自由切换，所有业务数据按店家完全隔离。

#### 规则说明

1. 所有业务数据（妹妹、套餐、顾客、订单、标签）均绑定对应店家，不同店家数据互不可见、互不影响
2. 当前切换到某店家时，新增的所有信息自动归属当前店家
3. 所有包含「归属店家」的字段仅做展示，不可编辑，禁止手动修改
4. 每个店家可单独配置**客服提成规则**（提成类型：比例/固定、提成数值），切换店家后自动应用对应规则

#### 数据隔离范围

妹妹信息、套餐配置、顾客档案、顾客标签、预约订单、提成数据、统计看板

------

### 2. 妹妹信息管理

#### 2.1 基础信息

| 字段     | 说明                   | 规则                                   |
| -------- | ---------------------- | -------------------------------------- |
| 妹妹ID   | 系统自动生成，唯一标识 | 创建后不可编辑                         |
| 妹妹姓名 | 妹妹真实姓名/花名      | 必填，同一店家内不可重复               |
| 状态     | 在职/休息/离职         | 默认在职，可选                         |
| 归属店家 | 关联对应店家           | 创建时自动带入当前店家，仅展示不可编辑 |

#### 2.2 提成规则配置

妹妹提成支持两种模式，可自由选择配置：

| 模式         | 计算规则                                    | 适用场景               |
| ------------ | ------------------------------------------- | ---------------------- |
| 比例模式     | 妹妹收入 = 订单金额 × 设置的百分比          | 适合按成交额抽成的场景 |
| 固定金额模式 | 妹妹收入 = 设置的固定数额（与订单金额无关） | 适合固定单价的场景     |

> 配置字段：提成类型（比例/固定）、提成数值

------

### 3. 套餐信息管理

#### 3.1 基础套餐库（店家级）

每个店家可独立维护自有套餐，无需硬编码套餐类型：

| 字段     | 说明               | 规则                             |
| -------- | ------------------ | -------------------------------- |
| 套餐编码 | 如SSS、SS、S       | 必填，同一店家内不可重复         |
| 套餐名称 | 如热恋、中恋、初恋 | 必填                             |
| 状态     | 上架/下架          | 默认上架，下架后不可用于新建订单 |
| 归属店家 | 关联对应店家       | 创建时自动带入，仅展示不可编辑   |

#### 3.2 妹妹套餐价格配置

同一个套餐在不同妹妹处的售价可不同，单独配置：

| 字段     | 说明                           | 规则                           |
| -------- | ------------------------------ | ------------------------------ |
| 关联妹妹 | 选择对应妹妹                   | 必填                           |
| 关联套餐 | 选择当前店家基础套餐库中的套餐 | 必填                           |
| 实际售价 | 该妹妹对应套餐的对外售价       | 必填，正整数                   |
| 归属店家 | 关联对应店家                   | 创建时自动带入，仅展示不可编辑 |

> 规则：同一妹妹+同一套餐的价格配置不可重复

------

### 4. 顾客信息管理

#### 4.1 顾客档案（核心）

用于统一关联顾客的所有账号，解决同一个顾客多账号的问题：

| 字段     | 说明                             | 规则                           |
| -------- | -------------------------------- | ------------------------------ |
| 顾客ID   | 系统自动生成，唯一标识           | 创建后不可编辑                 |
| 昵称     | 顾客常用称呼                     | 可选                           |
| 备注     | 其他补充信息（如偏好、特殊要求） | 可选                           |
| 归属店家 | 关联对应店家                     | 创建时自动带入，仅展示不可编辑 |

#### 4.2 顾客账号管理

一个顾客档案可绑定多个平台的账号，适配TG炸号迭代需求：

| 字段     | 说明                   | 规则           |
| -------- | ---------------------- | -------------- |
| 账号ID   | 系统自动生成，唯一标识 | 创建后不可编辑 |
| 平台类型 | 微信/TG                | 必填           |
| 平台账号 | 对应平台的用户ID       | 必填           |
| 关联顾客 | 绑定对应顾客档案       | 必填           |

> 规则：查询任意一个顾客账号，都可快速定位到对应的顾客档案，避免TG炸号后历史消费数据丢失；同一个顾客可绑定多个同平台账号（如多个TG号）

#### 4.3 顾客标签管理

支持自定义标签对顾客打标，方便后续分类运营：

| 字段     | 说明                                             | 规则                           |
| -------- | ------------------------------------------------ | ------------------------------ |
| 标签名称 | 自定义文本，如口臭、态度好、回头客、放鸽子       | 必填                           |
| 标签颜色 | 可选颜色（如红色代表负面标签、绿色代表正面标签） | 可选                           |
| 归属店家 | 关联对应店家                                     | 创建时自动带入，仅展示不可编辑 |

> 规则：一个顾客可绑定多个标签，标签为店家级资源，可自由新增/删除

------

### 5. 预约订单管理（核心业务）

#### 5.1 订单字段

| 字段     | 说明                                             | 规则                           |
| -------- | ------------------------------------------------ | ------------------------------ |
| 订单编号 | 系统自动生成，唯一标识                           | 创建后不可编辑                 |
| 关联顾客 | 选择顾客档案，自动带出该顾客的所有账号信息       | 必填                           |
| 关联妹妹 | 选择对应妹妹                                     | 必填                           |
| 套餐     | 选择当前店家的基础套餐                           | 必填                           |
| 套餐价格 | 自动带出当前妹妹对应套餐的实际售价，支持手动修改 | 必填，默认正整数               |
| 预约时间 | 顾客预约的服务时间                               | 必填                           |
| 订单状态 | 待确认/已完成/已取消                             | 默认待确认，可手动修改         |
| 归属店家 | 关联对应店家                                     | 创建时自动带入，仅展示不可编辑 |

#### 5.2 自动计算规则（订单生成时自动计算，结果固化）

订单保存时自动计算以下数据，计算结果永久固化，后续修改妹妹/套餐/提成规则均不影响历史订单数据：

| 计算项   | 计算公式                                                     | 说明                         |
| -------- | ------------------------------------------------------------ | ---------------------------- |
| 订单金额 | 填写的套餐价格                                               | 订单核心金额                 |
| 妹妹收入 | 若妹妹提成为比例：订单金额 × 妹妹提成比例；若为固定：直接取妹妹提成固定值 | 按该妹妹配置的提成规则计算   |
| 客服提成 | 若客服提成为比例：订单金额 × 当前店家客服提成比例；若为固定：直接取当前店家客服提成固定值 | 按当前店家的客服提成规则计算 |

#### 5.3 订单快照机制

订单生成时自动保存以下快照信息，避免后续修改基础信息导致历史数据统计错误：

- 妹妹姓名、妹妹提成规则
- 套餐名称、套餐价格
- 客服提成规则
- 顾客账号信息

------

### 6. 数据看板

切换店家后自动更新为该店家的统计数据，核心指标如下：

#### 6.1 订单统计

- 今日订单数、今日订单总金额
- 本月累计订单数、本月累计订单金额
- 已完成订单数、已取消订单数

#### 6.2 妹妹统计

- 当前店家在职妹妹数量
- 本月各妹妹接单量排名
- 本月各妹妹收入排名

#### 6.3 客服提成统计

- 本月累计客服提成总额
- 各妹妹订单贡献的客服提成明细

#### 6.4 顾客统计

- 当前店家总顾客数
- 本月新增顾客数

#### 6.5 标签统计

- 高频负面标签分布（辅助客服规避问题妹妹/顾客）

------

### 7. 通用规则

1. 所有业务数据的归属店家字段均为系统自动带入，仅展示不可编辑，禁止手动修改
2. 数据校验规则：
   - 同一店家内妹妹姓名不可重复
   - 同一店家内套餐编码不可重复
   - 同一妹妹+同一套餐的价格配置不可重复
   - 订单金额、套餐价格必须为正整数
3. 历史订单数据永久固化，不支持修改核心计算参数，若需调整需走特殊流程（如创建调账订单）

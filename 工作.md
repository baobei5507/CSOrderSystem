# 客服预约管理系统（Cloudflare D1版）

## 项目目标

开发一个单客服使用的预约管理系统。

核心功能：

- 多店家管理
- 妹妹管理
- 套餐管理
- 顾客管理
- 顾客标签管理
- 预约订单管理
- 自动提成计算
- 数据看板

系统主要运行在手机端。

设计优先级：

移动端 > 平板 > 桌面端

---

## stores

店家

字段：

```sql

name TEXT NOT NULL

service_commission_type TEXT NOT NULL
service_commission_value REAL NOT NULL

updated_at INTEGER

- fixed

## girls

```

# 自动计算规则

订单保存时自动执行。

------

妹妹收入

如果：

text



```
commission_type=percent
```

计算：

text



```
订单金额 × commission_value ÷ 100
```

如果：

text



```
commission_type=fixed
```

计算：

text



```
commission_value
```

------

客服提成

读取当前店家配置。

如果：

text



```
service_commission_type=percent
```

计算：

text



```
订单金额 × service_commission_value ÷ 100
```

如果：

text



```
service_commission_type=fixed
```

计算：

text



```
service_commission_value
```

------

# 订单快照规则

订单创建时自动生成快照。

后续修改：

- 妹妹名称
- 提成规则
- 套餐名称
- 套餐价格
- 客服提成规则

均不得影响历史订单。

统计全部基于订单快照。
